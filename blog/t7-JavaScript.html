TJavaScript and Ruby are both Object Oriented Programming languages. But the ways they define objects syntactically are different.

Ruby Hashes vs. JavaScript Objects

Ruby Hashes are similar to the creation of a JavaScript object. In Ruby a hash is a a list of key value pairs (see example below): 

 my_hash = { key1: value1, key2: value2}

JavaScript uses similar syntax for the creation of an object. An object in JavaScript is defined as shown below:

var myJavaScriptObject = {
      property1: value1,
      property2: value2

      behavior1: function() {
        this.property1 +=1;
      }
    };

Ruby classes vs. JavaScript constructor functions

Ruby classes are defined with a class keyword. Furthermore it is characterized by attributes and methods. JavaScript doesn't seem to have a formal class definition for its object. They syntax for a ruby class is written below: 

class My_object

      @instance_variable1
      @instance_variable2

      def initialize(value1, value2)
        @instance_variable1 = value1
        @instance_variable2 = value2
      end

      def method_behavior_1
        @instance_variable1 += 1
      end


    end

Looping in Ruby vs. Looping in JavaScript

There are some differences between Ruby and Java concerninc looping. Whereas Ruby uses .each JavaScript uses the for loop as shown below.

Ruby iteration example: 

my_array = [1,2,3, 4]

my_array.each { |element_of_array| puts element_of_array}

JavaScript iteration example: 

var my_array = [1, 2, 3, 4];

for(var index = 0; index < my_array.length ; index += 1)
      console.log(my_array[index]);
